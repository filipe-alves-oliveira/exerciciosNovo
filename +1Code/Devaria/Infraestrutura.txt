A base para o funcionamento da tecnologia
-A infraestrutura de tecnologia da informação é composta de recursos físicos e virtuais que suportam o fluxo, armazenamento, processamento e análise de dados

Infraestrutura não é o fim e sim o MEIO.

Principais Componentes de um computador

Processador
-Também conhecido como CPU # do gabinete. Dentro do gabinete tem a CPU - Unidade Central de Processamento
-Calcular e realizar tarefas determinadas pelo usuário e é considerado o cérebro do computador, peça mais rápida e que precisa de mais velocidade.
-o que define se ele pe bom ou não é os núcleos, processadores de 4 núcleos são os melhores q fazem o processamento de 4 coisas ao mesmo temp- Os núcleos de processamento são capazes de processar multitarefas

- Partes mais importantes do processador: 
-ULA (Unidade de calculo aritmetico +, *, -, /).
-Clock (Velocidade do processador, não adianta comprar um core i5 de 8 nucleos da geração 3, o clock dele será muito mais baixo que nas novas gerações, melhor comprar um processador com clock melhor e mais nucleos. 2.4 gigahertz, é 2.4 bilhoes de processamentos de calculos por segundos).
-Cache (ex.: calculadora HP - grava a % para fazer o calculo no final da conta, o cache é isso, guarda os dados q vão ser usados depois para que o processador nao precise colocar esses dados em outro lugar ele mesmo grava esses dados q vai usar rapidamente)
-Registradores (guardam regras, calculos aritmeticos que a ULA utiliza para fazer os calculos, Memoria de coisas aritmeticos.)
NÃO PRECISAMOS PROGRAMAR NADA DISSO, QM FAZ ESSE TRABALHO É O SO E O PROCESSADOR.

Ordem de importância dos componentes do computador para programação -> Memoria RAM, Processador, SSD.

Memória RAM
-É a memória temporária de um computador, SÓ UTILIZADA COM ENERGIA QDO PC TA LIGADO, armazena dados de forma temporaria, qdo abrimos o browser, programas, windows, td q está sendo executado está usando a memória RAM para executar. A memoria RAM é muito mais rápida q o disco rigido que é o HD e o SSD. Os programas utilizam essa memoria em execução para ter mais velocidade. Voce coloca várias coisas na MEMORIA RAM e o sistema vai utilizando essa memoria e qdo vc fecha o programa ele pega a memoria guarda no HD qdo necessario e o resto ele limpa a memória.
-Utilizada para o processador acessar os dados mais rápidos para fazer os cálculos
-É utilizada sempre quando os programas e serviços estão sendo executados.
-Dois tipos de memória: SDR e DDR, sendo que a versão 4 é a mais potente, ou seja, DDR3 mais potente que a 3 e hoje está na DDR5 lançada em 2021 e qtos GB tem capacidade de armazenar, geralmente compramos de 4gb, 8gb ou 16gb. Se fosse um back e front seria necessário minimo de 8gb


Hard Disk Drive
-O Hard Disk Drive é, ainda hoje, a principal forma de armazenar dados em um computador. Mesmo com a existência do SSD, o atual substituto do HD, eles reinam na maioria dos PCs de mesa e notebooks, voce grava as informações em um disco CD mesmo e usa essas informações gravadas qdo o sistema pede. Só que é diferente da memória RAM, pq qdo desligamos o PC não perde os dados ficam no HD ou SSD.

Solid Slate Drive
-Um SSD (Solid State Drive) é o substituto e concorrente do HDD, a peça traz uma série de melhorias, principalmente de velocidade, para a máquina, seja um desktop ou um notebook. Diferente do HD ele não utiliza disco utiliza uma memoria flash, um novo conceito de armazenamento, é uma evolução como exemplo: de disco vinil para CD.
A diferença
SSD # HDD - tempo de acesso, qdo vc acessa um arquivo no HD tem um tempo de 5.5 ms no SSD 0.1 ms, ou seja, muito mais rápido.


Placa de rede
-Responsável pela conexão a com a rede de computadores.
-Todos os dados que SAEM e ENTRAM no computador passam pela placa de rede, seja ela uma placa de rede sem ou com fio.
-As placas de redes podem transmitir em velocidades que variam de 10Mbps até 10 Gbps, e isso determina a performance que o computador tem na transmissão de dados com a internet por exemplo

Fonte
-Equipamento que alimenta a energia de um computador, notebook e servidores. É responsável por distribuir a energia necessária para cada peça dentro da máquina, fazendo com que todos tenham o funcionamento.
-Existem diversas potências de fonte e tudo depende qual equipamento ela vai alimentar, pois quanto mais potente seja o equipamento mais capacidade de energia a fonte tem que prover.
-Eficiência: Na hora de comprar é muito importante escolher uma bem avaliada pelo 80plus

Placa mãe
-É a peça do computador que interliga todas as outras mencionadas anteriormente, ou seja, ela é responsável pela conexão entre o processador, memória RAM, Fonte, placa de rede, placa de vídeo, HD, driver de CD, entradas USBs entre outros componentes.
-Além de interligar as peças a placa mãe trafega do com todos os dados entre as peças, então ela desenvolve um papel extremamente importante.
-Os modelos de diferenciam de pois cada placa tem suporte para peças com características diferentes e com as evoluções das peças as placas mães vão se adaptando. Processador antigo não encaixa em uma placa mãe nova. Encaixe de slots de memória. Placa q não aceita SSD, placa q aceita.

Placa de vídeo
-É o componente responsável por disponibilizar o vídeo no monitor, controlando e processando todos os gráficos para que o usuário possa visualizar em formato de imagem e vídeos.
-A GPU (Unidade de Processamento Gráfico) é a parte mais importante da placa de vídeo. É o processador responsável pela execução de cálculos e rotinas que resultam nas imagens exibidas no monitor.
-Qto melhor o GPU, mais processamento de imagem vou ter.

Redes de computadores

O que é redes?

É a conexão de equipamentos (computadores, notebooks, impressoras, celulares ....) para permitir a troca de informações entre eles, possibilitando assim a utilização dos sistemas.

Classificação de redes

-LAN
Mais conhecida como rede local, que é a interligação de equipamentos em um mesmo ambiente.
-MANMemória RAM
Rede metropolitana, ou seja, é uma interligação de equipamentos em um uma distância regional.
-WAN
Interliga equipamentos em longas distâncias, levando os dados para diversos países e regiões.

Como é distribuído os dados?

Redes moveis / Rede Residenciais -> ISP global -> ISP Regional <- Redes Corporativas.
ISP regional - empresas q tomam conta e controlam todo trafego de internet regional.
ISP global - entidade que recebem e trafegam os dados globalmente para tds paises.

Mas os dados trafegam como?

-Protocolo de redes
É uma bagunça a rede de computadores?
Como que as máquinas se entendem?
Protocolo são regras e normas utilizadas para a comunicação entre máquinas e equipamentos.
Vamos falar de IP, TCP, HTTP, FTP, SMTP, SSH e DNS.

-Protocolo HTTP e HTTPs*MAIS IMPORTANTE.
Protocolo utilizado para comunicação entre cliente e servidor
Toda comunicação HTTP funciona com uma requisição e uma resposta
A aplicação mais comum do HTTP é a navegação de sites nos Browsers, troca de dados entre sistemas e Mobile
Notebook -> Requisição -> Servidor -> resposta -> Notebook.

O que é esse PACOTE?
-Cabeçalho da mensagem: informações adicionais e o que está fazendo, GET, POST, PUT, Delete.
-Corpo da MSG: Conteudo da msg para o sevidor como: application.json, img/jpeg, text/plain, text/html.

-Método de requisição GET
Requisição do cliente para o servidor para OBTER alguma informação
CLIENTE -> Requisição de alguma informação -> servidor -> Resposta com a informação solicitada -> CLIENTE.

-Método de requisição POST
Requisição do cliente para o servidor para ENVIAR alguma informação para gravação
CLIENTE -> Requisição com dados para ser salvos -> servidor -> Resposta se houvesse sucesso no processamento -> CLIENTE.

-Método de requisição PUT
Requisição do cliente para o servidor para ATUALIZAR alguma informação
CLIENTE -> Requisição com dados para ser atualizado -> servidor -> Resposta se houvesse sucesso no processamento -> CLIENTE.

-Método de requisição DELETE
Requisição do cliente para o servidor para DELETAR alguma informação
CLIENTE -> Requisição com dados para ser excluido -> servidor -> Resposta se houvesse sucesso no processamento -> CLIENTE.

1xx: Informational (Informação) – Informa que a requisição foi atendida e está em processo - 100, 101, 102;
2xx: Success (Sucesso) – Requisição do cliente foi bem sucedida - 200, 201, 202;
3xx: Redirection (Redirecionamento) – Ação adicional que deve ser tomada - 300, 301;
4xx: Client Error (Erro no CLIENTE) – Avisa que o cliente fez uma requisição que não pode ser atendida - 400, 402;
5xx: Server Error (Erro no SERVIDOR) –Erro no servidor ao cumprir uma requisição válida - 500, 501.

Segurança HTTP/HTTPs
-Uso do SSL/TLS
Instala certificado digital no seu servidor e ele deixa de ser HTTP e vira HTTPs, seguro, ganhando autenticidade.
Essa prática faz com o que o HTTP vire HTTPS, sendo configurado um certificado para gerar autenticidade e criptografia. Empresas próprias fazem isso, emitem certificados para empresa.

-Uso de autenticação nas requisições
É uma forma de garantir que quem requisite tem autorização dos dados. Alguns protocolos mais utilizados: 
-No Auth
-Basic Authentication - usuario e senha para qm quiser utilizar o meu servidor. 
-oAuth2 (ex com Bearer token)

*Para que serve o modelo OSI e TCP IP?
Ele foi projetado para permitir a comunicação em qualquer tipo de rede, por isso funciona como um modelo de referência. Por outro lado, o modelo TCP/IP é uma implementação específica do modelo OSI voltada para a internet. Ele foi criado por conta da necessidade de padronização de outras tecnologias do stack de rede.

Protocolo TCP/IP
-Transmission Control Protocol
-Responsável pelas regras de transmissão dos dados para toda a internet
-Possui quatro camadas base: Aplicação -> Transporte -> Internet -> Acesso a Rede
-De baixo desse protocolo tão importante existem diversos outros como:
HTTP, SMTP, FTP, DNS

Diferença do modelo OSI(referencia de modelo de rede) versus TCP/IP(regras de protocolo TCP)
    OSI       x  TCP/IP
Camadas 7,6,5 x Aplicação         - camada de usuário que vemos
Trasnport     x Trasnport         - responsavel por mandar os dados da internet de um lado p outro.
Network       x Network           - endereçamento, identificação, onde está o protocolo IP
Camadas 1,2   x Network Interface - Acesso a Rede, parte física, placa de rede. Bit e byte, trasnferencia de dados analogicos.

* O modelo de arquitetura TCP/IP possui suas funções divididas em camada da mesma forma que o OSI. A diferença principal nestas estruturas é o número de camadas encontradas em cada modelo: no OSI encontramos 7 camadas, enquanto no TCP/IP somente 4: Aplicação, Transporte, Rede e Interface de rede.

-FTP
É um protocolo de transferência de dados, que em português significa Protocolo de Transferência de Arquivos
-Utilizado muito para enviar arquivos entre servidores; server e clientes. Ex.: img pinguim, enviado p server.
-O SFTP ou FTPS é uma forma mais segura (1 S antes SFTP e 1 depois FTPS)
Um utiliza SSH para autenticação e criptografia (o SFTP) e a outra forma utiliza SSL/TLS (FTPS) (certificado digital). O SFTP ou FTPS é uma forma mais segura. Utiliza certificado como o HTTPs para tornar mais seguro.
-Programas utilizados para conectar via FTP
O mais famoso deles é o FileZilla, mas você pode utilizar até mesmo o Windows Explorer

EMAILs - SMTP - POP3 - IMAP
ex.: Devaria me enviou um e-mail para ativar a minha conta, no back eu configuro uma autenticação/conexão com o server SMTP utilizando os protocolos SMTP, q é mandar e-mail.
SMTP - Protocolo de Transferência de Correio Simples
-Responsável pelo envio de e-mails

POP3 - Protocolo dos Correios
ex.: Tenho um servidor de e-mail, acesso o outlook e quero configurar o outlook na minha máquina como POP, utilizando o protocolo POP, eu baixo o e-mail e ele vem direto para minha máquina, diferente do IMAP.
-Responsável pelo recebimento de e-mails
o e-mail que é transferido desse protocolo é baixado para o ambiente local, retirando esse e-mail do servidor

IMAP - Protocolo de acesso a mensagem da internet
-Responsável pelo recebimento de e-mails
Todo o e-mail baixado por esse protocolo é preservado no servidor, outlook, mantendo o sincronismo das mensagens

SSH
Protocolo de rede criptográfico, uma forma de autenticar o server usando criptografia
-Canal seguro para conexão com servidores
Método comum utilizado para conexão remota com servidores, através de uma autenticação
-Envio de linhas de comando
Servidores de aplicações podem receber commandos via SSH de um administrador, como por exemplo inicial, pausar ou desligar um serviço
PUT - sistema q utiliza para conectar o sever SSH no windows.

DNS
Qdo entrar no site da Uol não preciso colocar o IP e já me direciona para o servidor da uol, por conta do DNS, responsavel por traduzir o IP para um nome amigavel o ip da uol, será www.uol.com.br 
Sistema de Nomes de Domínio
-Responsável pela gestão de nomes para computadores
Com o DNS você não precisa chamar um site, um serviço ou um computador pelo IP.

-O DNS não é uma gestão centralizada

Como funciona uma rede Lan?

INTERNET (rede pública) -> MODEM (entra na empresa atraves do modem e esse trasmiti os dados para o roteador) -> Roteador -> Firewall (protetor dos dados) -> Switch (distribui os dados pelos equipamentos) -> Access Point (mandar sinal Wireless para tds equipamentos que tem rede sem fio)
Switch -> Impressoras -> PC -> Notebooks.
Switch -> Servidores.

Modem
-A junção do Modulador-Demulador
-Transforma os sinais digitais em analógicos e vice-versa, HTTP.
-Responsável por pegar os pacotes e transformá-los em analógicos
-Geralmente são as operadoras que instalam e se responsabilizam pelos Modems

Roteador
-Responsável pela distribuição dos pacotes
-Verifica as melhores rotas para envio e recebimento
-Residências costumam utilizar roteadores com antenas Wireless
-Responsável por pegar os pacotes e transformá-los em analógicos
-Empresas maiores não tem o costume de utilizar Wireless junto com roteadores

Firewall
-Monitora todo o tráfego de saída e entrada de uma rede
-O firewall pode permitir/bloquear qualquer pacote de acordo com alguma característica
-Responsável por segurar/proteger ataques de hackres
-Firewalls podem estar em formato de software ou hardware embarcado com um software

Switch
-São os responsáveis por conectar todos os equipamentos de uma rede
-Existem os switches gerenciáveis e os não gerenciáveis
-São compostos de portas cabeadas

Access Point
-Dispositivo que permite equipamentos se conectarem sem fio a rede
-Uma forma prática e segura para escalar a rede sem necessidade de aumentar a infraestrutura
-Possuem controles como separação de redes de convidados até mesmo limitação de banda

Conceitos de cloud computing

Computação na nuvem

É a disponibilização de recursos tecnológicos pela internet com uma cobrança dos clientes através do uso do recurso.
Paga pelo serviço que utiliza.

IaaS – Infraestrutura como serviço (infraestructure as a service) - Alugar um serviço de infra como firewall..
PaaS – Plataforma como serviço (Plataform as a service) - comprar plataforma de mensageria, worpress, sms..
SaaS – Software como serviço (software as a service) - compra o software em nuvem ex.: spotify, compra e utiliza ele como serviço, vc nao tem ele na sua casa, nem um data center em algum lugar eles utilizam a amazon, nao tem data center proprio.

O Gartner (centro de pesquisa) prevê que, em 2025, 80% das organizações migrarão totalmente dos data centers locais para a hospedagem em nuvem.

Benefícios de utilizar Cloud

-Agilidade
Você tem um leque gigante de serviços e recursos disponíveis em poucos cliques.

-Elasticidade
Se seu negócio crescer e precisar de mais recursos tecnológicos no Cloud é possível escalar.

-Economia
Se bem escolhido e configurado. Os recursos que são utilizados serão de menor custo. Configurar a nuvem de acordo com sua necessidade.

-Geolocalização
A nuvem te permite ampliar o uso em diversas regiões.

Alguns de serviços em IAAS
-Servidores de armazenamento
Possibilidade de configurar servidores de arquivos de acordo com a demanda.
Storage - um monte de HD, SSD

-Segurança
É possível configurar Firewalls para ambientes empresariais e outras ferramentas de segurança.

-Servidores de usuários
Configurar todos os acessos de rede pelo Cloud como por exemplo o Active Directory (servidor de controle de usuario, senha, renovação de senha, e-mail de ativação) pode migrar ele para nuvem.

-Backups
Todos as aplicações, banco de dados e arquivos podem ser armazenados como Backup nos servidores Cloud
A lei LGPD não pode fazer backup fora do país.

Os principais provedores
Amazon Web Services(AWS), Microsoft Azure, Google Cloud.
